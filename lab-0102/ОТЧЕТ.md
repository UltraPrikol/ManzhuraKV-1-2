### Контрольные вопросы:
   - Что такое SSG (Static Site Generation) и как он реализован в вашем проекте?
      Static Site Generation — это генерация HTML-страниц во время сборки (build). 
      В проекте это реализовано через generateStaticParams, который заранее создает страницы для каждой статьи блога.
   - Как работает файловая маршрутизация в Next.js?
      Каждая папка внутри app/ становится путем в URL (например, app/about/ -> /about). Файл page.tsx внутри папки — это содержимое страницы.
   - Какие преимущества даёт использование `generateStaticParams`?
     Позволяет Next.js заранее знать список динамических путей. Это ускоряет загрузку (страницы уже готовы) и улучшает SEO.
   - В чём разница между `npm run dev` и `npm run build`?
      dev запускает сервер для разработки с "горячей" перезагрузкой кода. 
      build компилирует проект, оптимизирует код и готовит его к деплою (создает готовые HTML/JS файлы).
