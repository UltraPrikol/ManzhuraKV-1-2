## Теоретическая часть
Git — распределенная система контроля версий, созданная Линусом Торвальдсом. Позволяет
отслеживать изменения в коде, возвращаться к предыдущим версиям и организовать командную
работу.
GitHub — веб-сервис для хостига IT-проектов и их совместной разработки с использованием Git.
Основные понятия:
Репозиторий — хранилище вашего проекта и его истории изменений.
Коммит — фиксация изменений в репозитории с уникальным идентификатором (хешем).
Индекс (Staging Area) — промежуточная область, куда добавляются изменения перед
коммитом.
Ветка (Branch) — изолированная линия разработки. По умолчанию создается ветка main.
Статусы файлов: untracked, modified, staged.
Основные команды:
git init – инициализация нового репозитория.
git status – показывает состояние рабочего каталога и раздела проиндексированных
файлов.
git add – добавляет изменение из рабочего каталога в раздел проиндексированных файлов
("staging area").
git commit -m "message" – фиксирует проиндексированные изменения, сохраняя снимок
состояния проекта в истории версий.
git branch <branch_name> – создает новую ветку.
git checkout <branch_name> – переключается на указанную ветку.
git merge <branch_name> – вносит изменения из указанной ветки в текущую.
git remote add origin – привязывает локальный репозиторий к удаленному на GitHub.
git push -u origin <branch_name> – отправляет изменения из локального репозитория в
удаленный.

## Практическая часть
 Часть 0: Предварительная настройка (Аутентификация по SSH)
0102_lab_Система контроля и управления версий.md
Задание соответствует стр. 6-10 методички. Это обязательный шаг для удобной работы с GitHub
из Ubuntu.
1. Откройте терминал в Ubuntu.
2. Сгенерируйте пару SSH-ключей (алгоритм Ed25519 считается более современным и
безопасным):
ssh-keygen -t ed25519 -C "your_email@example.com"
На запрос о месте сохранения ключей нажмите Enter (ключи сохранятся в стандартной
папке ~/.ssh/).
Придумайте и введите надежный пароль (passphrase) для защиты вашего закрытого
ключа. Запомните его.
3. Запустите фоновый процесс ssh-agent и добавьте ваш приватный ключ в его управление:
eval "$(ssh-agent -s)" 
ssh-add ~/.ssh/id_ed25519 
Введите пароль от ключа при запросе.
4. Скопируйте содержимое вашего публичного ключа в буфер обмена:
cat ~/.ssh/id_ed25519.pub 
Выделите и скопируйте весь вывод команды, начиная с ssh-ed25519 ... и до конца.
5. Добавьте SSH-ключ в ваш аккаунт GitHub:
Перейдите на GitHub.com в настройки аккаунта (Settings -> SSH and GPG Keys).
Нажмите New SSH key.
В поле Title укажите понятное название (например, My Ubuntu Laptop).
В поле Key вставьте скопированный публичный ключ.
Нажмите Add SSH key.
6. Проверьте подключение:
ssh -T git@github.com
